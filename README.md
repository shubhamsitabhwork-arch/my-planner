| Feature | Description |
|---|---|

## 🏗 Architecture

```
User → React App (Vite) → Supabase API → PostgreSQL
         ↓                      ↓
      Vercel (web)          Auth (JWT)
      Capacitor (APK)       RLS Security
```

**Frontend:** React 18 + Vite — deployed on Vercel  
**Backend:** Supabase (Auth + REST API) — no custom server  
**Database:** PostgreSQL with Row Level Security  
**Mobile:** Capacitor wrapping the web app into an Android APK  

---

## 🗄 Database Schema

```sql
todos     (id, user_id, date, text, done, created_at)
schedule  (id, user_id, date, hour, content, created_at)
journal   (id, user_id, date, text, mood, created_at)
```

All tables have RLS enabled. Users can only access their own data.

## 🚀 Getting Started

### Prerequisites
- Node.js 18+
- A Supabase account (free)
- Android Studio (for APK builds only)

### 1. Clone and install
```bash
git clone https://github.com/YOUR_USERNAME/my-planner.git
cd my-planner
npm install
```

### 2. Configure Supabase
Edit `src/supabase.js` and replace with your project URL and anon key:
```js
const SUPABASE_URL = 'https://YOUR_PROJECT_ID.supabase.co'
const SUPABASE_ANON_KEY = 'your-anon-key-here'
```

### 3. Set up the database
Run this SQL in your Supabase SQL Editor:
```sql
create table public.todos (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users not null,
  date text not null, text text not null, done boolean default false,
  created_at timestamp with time zone default now()
);
create table public.schedule (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users not null,
  date text not null, hour integer not null, content text not null,
  created_at timestamp with time zone default now()
);
create table public.journal (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users not null,
  date text not null, text text, mood text,
  created_at timestamp with time zone default now(),
  unique(user_id, date)
);
alter table public.todos enable row level security;
alter table public.schedule enable row level security;
alter table public.journal enable row level security;
create policy "own todos" on public.todos for all using (auth.uid() = user_id);
create policy "own schedule" on public.schedule for all using (auth.uid() = user_id);
create policy "own journal" on public.journal for all using (auth.uid() = user_id);
```

### 4. Run locally
```bash
npm run dev
# Open http://localhost:5173
```

### 5. Build for web (Vercel)
```bash
npm run build
# Push to GitHub → Vercel auto-deploys
```

### 6. Build Android APK
```bash
npm run build
npx cap sync android
npx cap open android
# In Android Studio: Build → Generate APKs
```
